# رمز خطأ Messenger 10–1893063: الصفحات المقيّدة مؤقتاً من الإرسال

> **جديد** — تمت إضافته إلى مرجع رموز أخطاء منصة Messenger في 25 يونيو 2026.

وثّقت Meta رسمياً **رمز الخطأ `10 – 1893063`** في مرجع رموز أخطاء منصة Messenger. يُعاد هذا الخطأ من واجهة Send API عندما **تكون صفحة Facebook مقيّدة مؤقتاً من إرسال الرسائل** من قِبل Meta.

## ما هو هذا الخطأ؟

عندما تنتهك صفحة Facebook سياسات المراسلة الخاصة بـ Meta، أو يُبلَّغ عنها بسبب نشاط مشبوه، قد تقيّد Meta تلك الصفحة مؤقتاً من إرسال الرسائل. أي محاولة لإرسال رسالة عبر Messenger Send API أثناء سريان القيد ستفشل بالخطأ التالي:

```json
{
  "error": {
    "message": "Pages temporarily restricted from sending messages",
    "type": "OAuthException",
    "code": 10,
    "error_subcode": 1893063
  }
}
```

## تفاصيل الخطأ

| الحقل | القيمة |
|---|---|
| **رمز الخطأ** | `10` |
| **الرمز الفرعي** | `1893063` |
| **النوع** | `OAuthException` |
| **السبب** | الصفحة مقيّدة مؤقتاً من قِبل Meta |
| **هل يمكن إعادة المحاولة؟** | لا — يجب الانتظار حتى رفع القيد |

## كيفية معالجة هذا الخطأ في الكود

عند استقبال هذا الخطأ، يجب على تكاملك:

1. **تسجيله كقيد من المنصة**، وليس كخطأ في الكود
2. **تنبيه فريق العمليات** فوراً
3. **عدم إعادة المحاولة** — إعادة المحاولة لن تُجدي نفعاً أثناء القيد
4. **المراقبة** حتى رفع القيد قبل استئناف الإرسال
