# زر إكمال التدفق (Complete Flow) في قوالب واتساب

زر **"إكمال التدفق"** أو (Complete flow) (والمعروف برمجياً في الـ API كنوع `FLOW`) هو زر قوي جداً من أزرار الدعوة لاتخاذ إجراء (CTA) في قوالب رسائل واتساب. بدلاً من توجيه المستخدمين إلى موقع ويب خارجي أو طلب رد سريع تقليدي، يقوم هذا الزر بفتح **WhatsApp Flow** (تدفق واتساب تفاعلي) فوراً داخل واجهة المحادثة.

هذا يسمح للشركات بدمج النماذج التفاعلية، تدفقات الحجز، أو الاستبيانات بشكل أصلي تماماً (Native) داخل واتساب.

## 1. هيكل الـ API (JSON Payload)

لإرفاق زر إكمال التدفق في قالب عبر Meta Graph API، يجب تحديد نوع الزر كـ `FLOW` بداخل مصفوفة الـ `components` الخاصة بالقالب.

```json
{
  "name": "booking_flow_template",
  "language": "en_US",
  "category": "UTILITY",
  "components": [
    {
      "type": "BODY",
      "text": "Tap the button below to complete your booking."
    },
    {
      "type": "BUTTONS",
      "buttons": [
        {
          "type": "FLOW",
          "text": "Complete Booking",
          "flow_id": "123456789012345",
          "flow_action": "navigate",
          "navigate_screen": "START_SCREEN"
        }
      ]
    }
  ]
}
```

### المعطيات الأساسية (Key Parameters):
- `type`: يجب أن يكون حصراً `FLOW`.
- `text`: النص أو التسمية التي تظهر على الزر للمستخدم.
- `flow_id`: المعرف الفريد (ID) الخاص بـ WhatsApp Flow.
- `flow_action`: الإجراء الذي سيتم عند الضغط على الزر. عادةً يكون `navigate`.
- `navigate_screen`: معرّف (ID) الشاشة المحددة داخل الـ Flow JSON والتي يجب أن تفتح أولاً.

---

## 2. قيود الـ UI وقواعد التحقق (Limitations & Validations)

عند تصميم قالب يحتوي على زر `FLOW`، يجب عليك الالتزام التام بالقيود الصارمة التي تفرضها Meta لتجنب رفض القالب أو حدوث أخطاء وقت التشغيل (Runtime Errors).

<!-- preview -->
✅ صحيح (Valid)
- النص `text` الموجود على زر الـ FLOW يجب ألا يتجاوز **25 حرفاً**.
- الـ Flow المرتبط بـ `flow_id` يجب أن يكون في حالة **المنشور (PUBLISHED)**. لا يمكن الموافقة على قالب إذا كان يربط بـ Flow في حالة المسودة (Draft).
- يمكنك دمج زر `FLOW` مع أزرار ردود سريعة أخرى، ولكن يُسمح لك بوضع زر `FLOW` **واحد فقط** في كل قالب.

❌ خاطئ (Invalid)
- محاولة تمرير متغيرات ديناميكية مثل `{{1}}` بداخل معطى `navigate_screen` الخاص بزر الـ FLOW. أسماء الشاشات يجب أن تكون ثابتة ومكتوبة يدوياً (Hardcoded) في وقت إنشاء القالب.
- إضافة زريّ `FLOW` إلى نفس القالب.
- محاولة الربط بـ `flow_id` ينتمي إلى حساب أعمال واتساب (WABA) مختلف.

## 3. أفضل الممارسات للمطورين
1. **الاختبار (Testing):** قم دائماً بإنشاء نسخة مسودة (DRAFT) مكررة من الـ Flow لاختبارها. لا يمكنك اختبار إرسال قالب عبر الـ API باستخدام مسودة، لذا يجب نشر إصدار أولي (v1) من الـ Flow الخاص بك لمجرد الحصول على موافقة القالب، ثم استمر في التعديل باستخدام الـ Endpoint للبيانات الديناميكية.
2. **البدائل (Fallback):** فكّر دائماً فيما سيحدث إذا كان إصدار واتساب الخاص بالمستخدم قديماً ولا يدعم خاصية Flows. تقوم Meta تلقائياً بعرض رسالة "يرجى تحديث واتساب"، ولكن من الجيد برمجياً متابعة المستخدم برسالة عادية إذا لم يكمل الـ Flow خلال 24 ساعة.
