# رسائل الوسائط في واتساب

يعد إرسال الوسائط أمراً أساسياً للتواصل الغني في SEEN V2. يمكن إرسال الوسائط إما عن طريق توفير رابط عام (URL) أو عن طريق رفع الوسائط إلى خوادم Meta أولاً وتمرير الـ `media_id`.

## أنواع الوسائط والحدود

تفرض Meta حدوداً صارمة على أحجام الملفات والتنسيقات المدعومة. سيؤدي إرسال التنسيقات غير المدعومة أو الملفات ذات الأحجام الكبيرة إلى الرفض الفوري.

| نوع الوسائط | الحد الأقصى للحجم | التنسيقات المدعومة |
|------------|----------|-------------------|
| **الصوت (Audio)**  | 16 MB    | `audio/aac`, `audio/mp4`, `audio/mpeg`, `audio/amr`, `audio/ogg` (Opus فقط) |
| **المستند (Document)**| 100 MB  | أي نوع MIME صالح (مثل `text/plain`، `application/pdf`، `application/vnd.ms-excel`) |
| **الصورة (Image)**  | 5 MB     | `image/jpeg`, `image/png` |
| **الفيديو (Video)**  | 16 MB    | `video/mp4`, `video/3gp` (برنامج ترميز الفيديو H.264 وبرنامج ترميز الصوت AAC فقط) |
| **الملصق (Sticker)**| 100 KB   | `image/webp` (الأبعاد الدقيقة: 512x512) |

## المعلمات والحقول

عند إرسال الوسائط عبر الرابط (URL):

| الحقل | النوع | مطلوب | الوصف |
|-------|------|----------|-------------|
| `type` | string | نعم | `image`، `video`، `audio`، `document`، أو `sticker` |
| `[type].link` | string | نعم | رابط HTTP/HTTPS لملف الوسائط |
| `[type].caption` | string | لا | نص التسمية التوضيحية (بحد أقصى 1024 حرفاً، مدعوم للصور/الفيديو/المستندات) |
| `[type].filename`| string | لا | اسم ملف مخصص (فقط لنوع المستند) |

عند إرسال الوسائط عبر المعرف (ID):

| الحقل | النوع | مطلوب | الوصف |
|-------|------|----------|-------------|
| `[type].id` | string | نعم | المعرف المرجع من واجهة برمجة تطبيقات رفع الوسائط (Media Upload API) |

## قواعد التحقق

| القاعدة | ❌ يُرفض | ✅ يُقبل |
|------|-----------------|-----------------|
| تنسيق الصورة | `.gif` أو `.bmp` | `.jpg`, `.jpeg`, `.png` |
| حجم الصورة | صورة بحجم 6 MB | صورة بحجم 4.9 MB |
| تنسيق الفيديو | `.mkv` أو `.avi` | `.mp4` (ترميز H.264) |
| أبعاد الملصق | 500x500 | 512x512 بالضبط |
| رابط URL | يتطلب مصادقة أو إعادة توجيه | رابط مباشر يُرجع Content-Type صالح |

## معاينة حية — كيف تبدو

<!-- preview:accepted -->
### ✅ الاستخدام الصحيح — إرسال صورة عبر الرابط

```json
POST /v19.0/{PHONE_NUMBER_ID}/messages
Authorization: Bearer {ACCESS_TOKEN}

{
  "messaging_product": "whatsapp",
  "recipient_type": "individual",
  "to": "{{CUSTOMER_PHONE}}",
  "type": "image",
  "image": {
    "link": "https://seen-v2.example.com/assets/promo.jpg",
    "caption": "Check out our latest summer collection! ☀️"
  }
}
```

**لماذا ينجح هذا:** الرابط هو رابط مباشر لملف `.jpg`، والتسمية التوضيحية أقل من 1024 حرفاً.

<!-- preview:rejected -->
### ❌ خطأ شائع — مفتاح نوع الوسائط خاطئ

```json
{
  "type": "image",
  "media": {
    "link": "https://example.com/image.png"
  }
}
```

**الخطأ:** `Error 100: Invalid parameter` — يجب أن يتطابق المفتاح داخل JSON مع الـ `type`. إذا كان `type` هو `"image"`، يجب أن يكون الكائن `"image": {...}`، وليس `"media": {...}`.

<!-- preview:rejected -->
### ❌ خطأ شائع — تجاوز حد الحجم

```json
{
  "type": "video",
  "video": {
    "link": "https://example.com/huge-presentation.mp4"
  }
}
```

**الخطأ:** إذا تجاوز الملف الموجود في الرابط 16MB، ستحاول Meta تنزيله، وستفشل في التحقق من الحجم، وسترسل خطأ webhook بشكل غير متزامن يشير إلى فشل التنزيل بسبب حدود الحجم.

## واجهة برمجة تطبيقات رفع الوسائط (بديل للروابط)

إذا كنت لا ترغب في كشف الوسائط عبر روابط عامة، قم برفعها إلى Meta أولاً.

1. **الرفع:**
```bash
curl -X POST "https://graph.facebook.com/v19.0/{PHONE_NUMBER_ID}/media" \
  -H "Authorization: Bearer {ACCESS_TOKEN}" \
  -F "file=@/path/to/local/file.pdf" \
  -F "type=application/pdf" \
  -F "messaging_product=whatsapp"
```
*يُرجع:* `{"id":"1234567890"}`

2. **الإرسال:**
```json
{
  "type": "document",
  "document": {
    "id": "1234567890",
    "filename": "SEEN_Invoice.pdf"
  }
}
```

## ملاحظات التنفيذ في SEEN V2

- **الرابط (URL) مقابل المعرف (ID):** فضل دائماً استخدام `link` للأصول الثابتة (الشعارات، لافتات التسويق) واستخدام `id` للأصول الديناميكية/الحساسة (الفواتير، التقارير) التي لا ينبغي أن تكون متاحة للجمهور على شبكة توصيل المحتوى (CDN) الخاصة بك.
- **دعم GIF:** لا يدعم واتساب تنسيقات الوسائط GIF محلياً في واجهة برمجة تطبيقات الأعمال (Business API). يجب عليك تحويل ملفات GIF إلى `.mp4` وإرسالها كـ `video`.

## قائمة التحقق من الجودة (QA)

- [ ] تنفيذ التحويل التلقائي للأنواع الشائعة غير المتوافقة (مثل تحويل `.gif` إلى `.mp4`).
- [ ] التحقق من أن أسماء ملفات المستندات تتضمن الامتداد الصحيح ليتمكن واتساب من عرض الأيقونة المناسبة.\n\n
